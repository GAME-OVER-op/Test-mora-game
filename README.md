# GameSpace

## v15 — настоящая причина лага: кодек музыки

### Причина
Лаги были связаны не с крутящимся элементом, а с фоновой музыкой `bgm.ogg` (Vorbis/OGG):
- на части устройств Vorbis декодируется программно → нагрузка → лаги, когда музыка включена;
- в некоторых приложениях/устройствах этот кодек вообще молчит.

### Решение
- Музыка перекодирована в **AAC-LC (.m4a, 44.1кГц, stereo, 160k)** — этот кодек декодируется аппаратно практически на всех Android.
- Исходный `bgm.ogg` удалён из проекта.
- Плеер устойчивее: обёрнут в try/catch + `setOnErrorListener` (ошибка декода не ломает экран).

### Крутящийся элемент (вентилятор)
- Убран эффект «дыхания» (пульсации scale). Вентилятор теперь только равномерно вращается.
- Вращение — GPU-трансформ (`View.rotation`, hardware layer), без покадрового `invalidate()`.

### Не трогали
- `start_animation.mp4` — оригинал, без перекодировки.
